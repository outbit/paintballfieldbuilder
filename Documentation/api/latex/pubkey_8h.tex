\subsection{Detailed Description}
This file contains helper classes/functions for implementing public key algorithms.

The class hierachies in this .h file tend to look like this: 
\begin{DoxyPre}
                  x1
                 / $\backslash$
                y1  z1
                 |  |
            x2<y1>  x2<z1>
                 |  |
                y2  z2
                 |  |
            x3<y2>  x3<z2>
                 |  |
                y3  z3
\end{DoxyPre}

\begin{DoxyItemize}
\item x1, y1, z1 are abstract interface classes defined in \hyperlink{cryptlib_8h}{cryptlib.h}
\item x2, y2, z2 are implementations of the interfaces using \char`\"{}abstract policies\char`\"{}, which are pure virtual functions that should return interfaces to interchangeable algorithms. These classes have \char`\"{}Base\char`\"{} suffixes.
\item x3, y3, z3 hold actual algorithms and implement those virtual functions. These classes have \char`\"{}Impl\char`\"{} suffixes.
\end{DoxyItemize}

The \char`\"{}TF\_\-\char`\"{} prefix means an implementation using trapdoor functions on integers. The \char`\"{}DL\_\-\char`\"{} prefix means an implementation using group operations (in groups where discrete log is hard).
